python编程命名规范

1. 模块命名

  • 模块尽量使用小写命名，首字母保持小写，尽量不要用下划线（除非多个单词，且数量不多的情况）
  • 因为很多模块文件存在与模块名称一致的类，模块采用小写，类采用首字母大写，这样就能区分开模块和类

2. 类命名

  • 类名使用驼峰（CamelCase）命名风格，首字母大写，私有类可用一个下划线开头

3. 函数命名

  • 函数名(包括类成员函数)一律小写，如有多个单词，用下划线隔开
  • 类内以单下划线（_）开头的函数表示的是protected类型的函数。只允许类本身与子类进行访问。
  • 类内以双下划线（__）开头的函数表示的是private类型的函数，只运行类本身进行访问。

4. 变量命名

  • 变量名尽量小写，如有多个单词，用下划线隔开；
  • 类内以单下划线（_）开头表示的变量是protected类型的变量。只允许类本身与子类进行访问。
  • 类内以双下划线（__）开头表示的变量是private类型的变量，只运行类本身进行访问。
  • 常量采用全大写，如有多个单词，是用下划线隔开

5. 编程（代码）规范

1. 空行

  • 空一行：用于类成员函数之间，或者用于区分不同逻辑块
  • 空两行：类与类，类与函数，函数与函数之间

2. 换行

  • 反斜杠
  • 字符串
  • 二元运算符

3. 缩进

  • 没有子模块，一层缩进
  • 有子模块
