2 GCC

2.1 什么是GCC

  • GCC 原名为 GNU C语言编译器（GNU C Compiler）
  • GCC（GNU Compiler Collection，GNU编译器套件）是由 GNU 开发的编程语言
    译器。GNU 编译器套件包括 C、C++、Objective-C、Java、Ada 和 Go 语言前
    端，也包括了这些语言的库（如 libstdc++，libgcj等）
  • GCC 不仅支持 C 的许多“方言”，也可以区别不同的 C 语言标准；可以使用命令行
    选项来控制编译器在翻译源代码时应该遵循哪个 C 标准。例如，当使用命令行参数
    -std=c99 启动 GCC 时，编译器支持 C99 标准。

2.2 GCC工作流程

2.3 gcc和g++的区别

gcc 和 g++ 都是GNU(组织)的一个编译器。

  • 编译阶段：gcc 把 .c 文件当成 C程序，把 .cpp 文件当成 C++ 程序；g++ 则都认为是 C++ 程序。
  • 链接阶段：gcc 命令不能自动和 C++ 程序使用的库联接，所以通常使用 g++ 来完成联接。

2.4 GCC常用参数选项

gcc编译选项 说明
-E 预处理指定的源文件，不进行编译
-S 编译指定的源文件，但是不进行汇编
-c 编译、汇编指定的源文件，但是不进行链接
[file1] -o [file2] 将文件 file2 编译成可执行文件 file1
-I directory 指定 include 包含文件的搜索目录
-g 在编译的时候，生成调试信息，该程序可以被调试器调试
-D 在程序编译的时候，指定一个宏
-w 不生成任何警告信息
-Wall 生成所有警告信息
-On n的取值范围：0~3。编译器的优化选项的4个级别，-O0表示没有优化，-O1为缺省值，-O3优化级别最高
-l 在程序编译的时候，指定使用的库
-L 指定编译的时候，搜索的库的路径
-fPIC/fpic 生成与位置无关的代码
-shared 生成共享目标文件，通常用在建立共享库时
-std 指定C方言，如:-std=c99，gcc默认的方言是GNU C

3 静态库和动态库

库可以简单理解为将一些源文件编译生成的 .o 文件打包，供给其它程序使用。一般是可以直接拿来用的变量、函数或类。

库文件有两种：静态库和动态库（共享库）。区别是：静态库在程序的链接阶段被复制到了程序中；动态库在链接阶段没有被复制到程序中，而是程序在运行时由系统动态加载到内存中供程序调用。

库的好处：

  1. 代码保密；
  2. 方便部署和分发。

3.3 静态库和动态库的对比

静态库、动态库区别来自链接阶段如何处理，链接成可执行程序。分别称为静态链接方式和动态链接方式。

静态库加载更快，但如果多次调用则需要多次加载，浪费资源，更新、部署、发布麻烦；动态库加载慢，但可以实现进程间资源共享（共享库），更新、部署、发布简单，可以控制何时加载动态库。

4 Makefile

一个工程中的源文件很多，不可能在终端中一个个执行。可以将编译命令按一定规则写入 Makefile 文件中，达到自动化编译的目的。一旦写好，只需要一个 make 命令，整个工程完全自动编译，极大的提高了软件开发的效率。

4.1 文件命名和规则

命令在在执行之前，需要先检查规则中的依赖是否存在：

  • 如果存在，执行命令
  • 如果不存在，向下检查其它的规则，检查有没有一个规则是用来生成这个依赖的，如果找到了，则执行该规则中的命令

检测更新，在执行规则中的命令时，会比较目标和依赖文件的时间：

  • 如果依赖的时间比目标的时间晚，需要重新生成目标
  • 如果依赖的时间比目标的时间早，目标不需要更新，对应规则中的命令不需要被执行

5 GDB调试

GDB 是由 GNU 软件系统社区提供的调试工具，同 GCC 配套组成了一套完整的开发环境，GDB 是 Linux 和许多类 Unix 系统中的标准开发环境。

一般来说，GDB 主要帮助你完成下面四个方面的功能：

  1. 启动程序，可以按照自定义的要求随心所欲的运行程序
  2. 可让被调试的程序在所指定的调置的断点处停住（断点可以是条件表达式）
  3. 当程序被停住时，可以检查此时程序中所发生的事
  4. 可以改变程序，将一个 BUG 产生的影响修正从而测试其他 BUG

vscode 现在已经很强大，基本上都可以用 vscode 代替 GDB。

  • -g 选项：确保生成的可执行程序能找到源代码
  • -Wall 选项：在尽量不影响程序行为的情况下选项打开所有warning，也可以发现许多问题，避免一些不必要的 BUG

6 文件IO

系统中的各种共享资源都由操作系统统一管理，因此，在用户程序中，凡是与资源管理有关的操作（如文件管理，IO传输等），都必须通过系统调用方式提出请求，并有操作系统代为完成。

所以 C 库文件 IO 函数均是系统调用 Linux 系统 IO 完成的。

Linux 中所有硬件都可以当作文件来对待，所以学习文件管理对 Linux 开发很重要。

6.1 文件描述符

进程在内存中执行，因为虚拟地址空间技术，用户不需要考虑进程在内存中的实际位置。虚拟地址空间的大小与CPU指令位数有关，对于32位系统，CPU最大支持4G内存，所以虚拟地址空间大小也就是4G。

每个进程都有一个PCB，PCB中有一个文件描述符表，用于管理进程执行所用到的文件。
